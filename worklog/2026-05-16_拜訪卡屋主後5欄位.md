# 2026-05-16 拜訪卡片屋主後加 5 個物件資訊欄位

## 【背景】

user 在「委託」tab → 拜訪物件地 看拜訪卡片時，卡片表頭只有「物件名 + 屋主：蔡先生 + 手機」。出門拜訪前想一眼看到該物件的關鍵資訊（現況、坪數、開價、格局），以及一鍵點進社區網頁，不用再開 Notion 翻。

需求：屋主名稱後面接 5 個標籤，對應 Notion 開發 DB：
1. 現況（自住/空屋）
2. 坪數
3. 開價
4. 格局
5. 「社區」兩字當超連結 → 網頁 URL

## 【做了什麼】

### 1. `app/api/dev/route.ts` — 補讀「現況」

- 查 live Notion 開發 DB schema 確認真實 property：`現況` 是 `select`、options 只有 `[空屋, 自住]`（坪數/開價/格局/網頁 4 個 API 早就有 `area`/`price`/`layout`/`web`，不用動）
- `DevPropertyData` interface 加 `occupancy?: string`
- `mapPage` 加 `occupancy: extractSelectValue(props['現況']?.select) || undefined`
- PATCH 不動（這次只讀不寫）

### 2. `app/marketing/page.tsx` — 拜訪卡片表頭加 5 標籤

- 位置：拜訪卡片表頭（「頭：物件名 + 屋主」那塊、約 2666 行「屋主：」span 後面），插一個 `flex gap-1.5 flex-wrap` 容器，內含 5 個 badge
- 只動這張**拜訪卡**（委託 tab 的 visit sub-tab），行銷頁 / 追蹤委託左右欄 / 其他「屋主：」render 完全沒碰
- 每個 badge **有值才顯示**（`{prop.xxx && ...}`），空的不出現
- 現況高亮配色：`空屋`→emerald 高亮綠、`自住`→orange 高亮橘、其他值→灰底（fallback，理論上不會有）
- 坪數/開價自動補單位：`endsWith('坪'/'萬')` 才不重複補，純數字 `54`/`1388` → 顯示 `54坪`/`1388萬`
- 格局原樣顯示（user 寫的 `3/2/2` 是示意、實際 Notion 是 `四房平車` 這種自由文字）
- 「社區」是 `<a href={prop.web} target="_blank">`、靛藍底 hover 變亮

### 驗證

- live Notion DB schema + 8 筆真實資料抽樣，確認欄位名/型別/內容格式（零腦補欄位名）
- dev server `npm run dev`、帶 crm-auth cookie 觸發編譯：`/marketing 200`、`/api/dev 200`、29 筆、`occupancy` key 已進、樣本「河堤京城一期」= 空屋/54/1388/四房平車/sinyi url
- user local 預覽逐項確認 OK（含現況高亮綠/橘）

## 【為什麼這樣做】

- **不腦補欄位名**：第一直覺以為「現況」可能不存在（初始 `create-new-dbs.js` schema 沒有它），實際是後來手動加的。照紀律先查 live Notion DB schema 才動 code，避免又踩 `[[feedback-no-invent-selectors-or-ids]]`
- **只動拜訪卡不全域**：codebase 有 5 處 render「屋主：」，user 要的是拜訪卡這一張（截圖明確）。全改會污染行銷頁等其他 view、違反北極星「不堆雜訊」
- **坪數/開價防呆補單位**：真實資料是純數字、但不排除某筆已帶單位；`endsWith` 判斷避免「54坪坪」
- **現況只認兩值高亮**：user 拍板維持「空屋/自住」兩種就好，第三種選項先不預想配色（出現會是灰底、屆時再補一行）

## 【下次接手要注意】

- **新物件自動出現、零 code 改**：`/api/dev` 是 `force-dynamic` 即時讀 Notion + badge 條件渲染。新物件只要 (a) 在同一個 `NOTION_DEV_DB_ID` DB (b) 欄位同名 (c) 進得了拜訪卡列表（有對應 `待辦` + active 狀態），重新整理就自己出現
- **「現況」select 加第三個 option**（如「出租」）：會顯示但**灰底不高亮**，要在 `page.tsx` 拜訪卡 occupancy 那段三元判斷補一條色（user 已知此限制、拍板先不做）
- **Notion 欄位被改名 = 抓不到**：`現況/坪數/開價/格局/網頁` 任一改名，對應 badge 靜默消失（不會報錯）
- 沒動 schema、沒寫腳本、沒碰 PATCH，純讀取 + UI
