# 2026-05-16 行銷詳情頁 重要大事/待辦 收斂為純檢視（輸入收上方公告欄）

## 【背景】

規劃討論：user 觀察上方公告欄（近期重要事項+待辦事項、會「將記錄到：選定客戶」）跟各別客戶詳情頁「最新進度」tab 的 重要大事+待辦事項，**兩處都能輸入 = 重覆**。user 提案：詳情頁只展示、不放輸入框，要新增就用上方公告欄。

CC 結構化 review（紀律 #4）：grep 確認詳情頁已有完整清單顯示（含完成/刪除），提案是小手術不是從零做；verdict = 方向對，但修一個關鍵點 → 「只展示」不該等於純唯讀，**完成勾選/劃線/刪除是檢視動作要保留**（且連動今日做的 Notion 內文劃線），移除的只是「新增 input」。user 拍板：保留完成/刪除、只拿輸入框。

## 【做了什麼】

`app/marketing/page.tsx` 單檔（最新進度 tab）：

- **重要大事**：移除 `<input>新增重要大事` + `+` 鈕；標題列改為 h3 + 極淡提示「＋ 新增請用上方公告欄」。下方清單 + 「完成」鈕**原樣保留**
- **待辦事項**：移除 `<input>新增待辦` + `DateTimePopover`(日期picker) + `+` 鈕 + `{newTodoDate && ...}` Calendar 提示行；標題列同上加提示。下方清單 + 勾選完成（含逾期/今日標色 + 劃線動畫）+ 刪除 **原樣保留**
- **死碼清除**：移除因此變未使用的 `handleAddImportant` / `handleAddTodo` 兩函式 + `newImportantText/newTodoText/newTodoDate/newTodoTime` 4 個 useState + `handleSelectClient` 內對應 4 個 setter reset 行
- 之前進度、目前進度(洽談 quick-log)、帶看記錄卡、上方公告欄 **全程未動**

新增動線：選客戶 → 上方公告欄會「將記錄到：該客戶」（`handleSelectClient` 仍 `setQuickImportantBound/setQuickTodoBound(true)` 自動綁），於上方輸入即可。

## 【為什麼這樣做】

- **輸入單一入口**：上方公告欄 = 捕捉/今日；詳情頁 = 檢視該客戶。回到系統原本該有的分工、符合北極星「不堆輸入雜訊」
- **保留完成/刪除**：檢視某客戶時最高頻動作是「打勾完成」，這是檢視不是輸入；且完成會連動 Notion 內文劃線（同日 A 功能）。拿掉會更糟
- **加極淡提示**：解發現性盲點（拿掉輸入框後怕忘記去哪新增），`text-[11px] text-slate-600` 不搶視覺
- **順手清死碼**：不留 dead handler/state 進 commit
