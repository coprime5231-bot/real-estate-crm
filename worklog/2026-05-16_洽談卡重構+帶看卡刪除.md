# 2026-05-16 Phase C：洽談卡重構 + 帶看卡加刪除

## 【背景】

接續同日 A（重要/待辦回寫內文）、B（之前進度全歷史+滑軌+置底）。Phase C 原暫緩、本輪 user 拍板開做。需求：

1. 洽談卡後面加「重要(亮橘圓點)」+「取消(暗灰圓點)」、像帶看卡收合一樣用圓點
2. 洽談寫完送出 → 預設出現像帶看卡的**收合**卡片
3. 內容點了能編輯，編輯後**回寫至 Notion 內文**
4. 按「重要」→ 高亮橘框 + 放大 + **置頂（次頂）**；按「取消」→ 收合
5. 排序：帶看卡(喜歡置頂) → 洽談卡(重要次頂) → 其餘按日期
6. （C 通過後追加）帶看卡圓點後也加刪除鈕 + 🗑、對齊洽談卡

user 拍板：動正式 PG（加 2 欄）、舊洽談「編輯同步 Notion」只往前生效（同 A 模式）。

## 【為什麼這樣做】

- **page.tsx 全程不動**：洽談/帶看卡都是獨立元件、tab 只 mount `ClientViewingsTab`。改元件不碰 2700 行的 page.tsx → 風險最小
- **三態鏡像 ViewingCard**：既有 disliked 收合/liked 展開模式成熟、洽談平行套用、認知一致、不發明新互動
- **存 block id 不靠文字比對**：同 A 拍板。編輯時據 `notion_block_id` 精準改那行
- **best-effort 全程**：Notion 失敗只 console.error 不擋 PG 主流程（沿用 quick-log/帶看寫回慣例）
- **帶看 DELETE 鏡像 conversations DELETE**：同 PG 表結構、同錯誤處理，零新模式
