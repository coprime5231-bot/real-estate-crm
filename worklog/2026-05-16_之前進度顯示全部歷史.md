# 2026-05-16 之前進度卡：固定高度可捲 + 顯示全部歷史（修後端 slice(-3)）

## 【背景】

user 要「之前進度」卡放大到約 5 行、可上下捲動看歷史。原始需求講「左側滑軌」、預覽時 user 更正為**右側滑軌**（預設）。

預覽中 user 回報「以前的進度記錄看不到」。CC 一開始臆測「之前進度顯示內文全部段落」是**講錯**——查 code 發現真因：`app/api/clients/[id]/blocks` GET 第 33 行 `slice(-3)` 把資料**寫死只回最新 3 筆**，舊的在後端就被砍掉。前端高度/滑軌怎麼改都沒意義（資料源頭只有 3 筆）。

## 【做了什麼】

2 檔：

### `app/api/clients/[id]/blocks/route.ts`（真正的修正）

- 移除 `const last3 = textBlocks.slice(-3).reverse()` → 改回傳**全部** text block（`[...textBlocks].reverse()`、最新在上）
- `notion.blocks.children.list` 加**分頁迴圈**（`while (has_more)` 帶 `start_cursor`）：舊客戶內文常 > 100 block、原本只抓第一頁 100 筆、且最新的在後面頁 → 不分頁會漏最新。現在抓完整頁
- POST（新增進度）未動

### `app/marketing/page.tsx`（③ 之前進度區塊）

- 清單外包一層固定高度可捲容器：`overflow-y-auto pr-1` + `style={{ maxHeight: '10rem' }}`（約 5 行單行高度、`pr-1` 讓內容不貼滑軌）
- 滑軌**右側**（預設、`overflow-y-auto` 本來就在右）。中途曾用 `dir=rtl/ltr` 做左滑軌、user 更正後拿掉
- 排序未動（沿用既有 `createdTime` 升冪、舊在上新在下）；已告知 user 可選改最新在上、user 此輪未要求

## 【為什麼這樣做】

- **真因在後端不在前端**：`slice(-3)` 是 hard cap，UI 改高度/滑軌對「只有 3 筆」無效。先 grep 資料源頭才動，沒繼續盲調前端（紀律：user 對自身系統觀察為準、CC 臆測要先 grep 驗）
- **分頁抓全部**：user 明確要「看得到舊的、沒東西被藏」；活躍客戶內文（洽談 log 累積）易 > 100 block，不分頁會漏 → 必須 paginate
- **B 獨立 push 不等 C**：B 已驗證、獨立、2 檔；C（洽談卡重構）大改+含 PG migration+設計未定、暫緩。比照同日 A 獨立 push（73d775f 一次 RUNNING）成功模式，避免綁包重演今早 build-fail 雷型。user 拍板
