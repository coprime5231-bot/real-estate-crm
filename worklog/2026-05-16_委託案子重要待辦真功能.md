# 2026-05-16 委託頁 追蹤/委託 案子 重要事項/待辦 真功能

## 【背景】

行銷詳情頁「重要/待辦收斂為純檢視」上線後，user 要委託頁 追蹤/委託 比照。探線發現**委託那兩塊是從沒接通的 skeleton**（disabled 假 input、待辦永遠空），不是行銷那種小調整。

關鍵資料模型釐清（data-first 探明）：
- 委託頁選的「案子」= live 開發 DB (`NOTION_DEV_DB_ID`=30156ff9) 頁
- 重要/待辦 DB 的 `🤩 追蹤與委託` relation **指向 `28d56ff9`=「開發-archive」死封存 DB**（對不到 live 案子）→ 此路死
- **活路**：live 開發 DB 自己有單向 forward relation `本周重要大事 → IMPORTANT_DB`、`待辦事項 → TODO_DB`。從案子這側讀這兩個關聯就能拿到該案子的重要/待辦
- 抽樣 5 個追蹤/委託案子這兩關聯**全空** → 只往前生效（同 A/B/C），舊案子不回填
- 開發 DB `重要事項` rich_text 欄 = user 拿來記屋主聯絡備註（黃南碩 R121…），**非結構化清單、不動**

user 拍板：委託案子當「人」一樣處理（社區名只是 user 認案子的標籤、要經營回報的是屋主）；分兩階段、每階段預覽；**不做刪除**、**不做完成劃線記錄**（以後再說）；新增 input 留委託頁（上方公告欄只認買方）。

## 【做了什麼 — Stage 1：能看 + 能完成】

- 新 `app/api/dev/[id]/items/route.ts` GET：retrieve 開發頁 → 讀 `本周重要大事`/`待辦事項` relation id → 逐筆 retrieve 回 `{id,title}`；important 濾掉 `優先级=DONE`、todos 濾掉 `待辦=true`（對齊行銷只回未完成）
- `app/marketing/page.tsx`：
  - 加 state `entrustImportantItems` / `entrustTodos`
  - 加 useEffect：`selectedPropertyId` 變動 → fetch `/api/dev/[id]/items`、null 則清空（鏡像 client fetch 模式）
  - 加 `handleCompleteEntrustImportant`（PATCH `/api/important-items/[id]` `{status:'done'}`）+ `handleCompleteEntrustTodo`（PATCH `/api/clients/todos/[id]` `{todoFlag:true}`）；都只更新委託清單
  - 換掉 ①重要事項 / ②待辦事項 skeleton：移除 disabled 假 input+鈕；保留 `selectedProperty.important` ⚠ 屋主備註（whitespace-pre-wrap、existing 不動）；接結構化清單 + 「完成」；加淡字「（新增功能下一階段開放）」
- ④目前進度 / 經營記錄卡 skeleton **不動**（範圍外）；新增/刪除/完成劃線 **不做**（Stage 2 / 拍板不做）

## 【為什麼這樣做】

- **走開發頁 forward relation 不走 item→archive**：item 側 `🤩追蹤與委託` 指死封存 DB，唯一活路是案子側的 `本周重要大事`/`待辦事項`。新端點專責解這兩關聯
- **複用既有完成端點**：`/api/important-items/[id]`、`/api/clients/todos/[id]` 都按 item id 操作、無買方依賴，委託直接複用零新風險（Phase A/C 的 body 劃線因委託 item 無存 內文BlockID → 自動 no-op、無害）
- **保留屋主備註 ⚠**：那是 user 既有真實資料、範圍外、不刪
- **Stage 1 獨立 push**：已驗證、獨立、可上線；Stage 2「能不能寫單向關聯」是已知未知 → 不綁一包（同 session A/B/C 一貫）
