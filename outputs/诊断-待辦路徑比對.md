### 次要發現（診斷備忘）

1. **Next.js 路徑重疊但應該不是根因**：`app/api/clients/todos/route.ts` 跟 `app/api/clients/[id]/todos/route.ts` 內容**完全相同**（GET/POST）。Next.js static segment (`todos`) 優先於 dynamic (`[id]`)，所以：
   - `GET /api/clients/<uuid>/todos` → dynamic 版
   - `GET /api/clients/todos` → static 版（如果有人這樣打）
   - `PATCH /api/clients/todos/<uuid>` → `[todoId]/route.ts`（綠色 PATCH 實際打這裡）
   路徑衝突不影響 PATCH 分派。
2. **兩條路徑共用相同 Notion DB**（env `NOTION_TODO_DB_ID`）、相同 page IDs、相同 `notion.pages.update` 寫入點。
3. **UI render 都在同一個檔案** `app/marketing/page.tsx`（不是獨立 modal component）。這推翻了第十三場「綠色路徑是獨立 component / API」的假設。

## 根因假設

### ✅ 選定：**A — 兩條路徑共用 notion 寫入點**

**證據**：PATCH URL、handler、notion call site **完全一樣**。只有 UI handler 名稱不同（`handleAnimatedToggleTodo` vs `handleToggleTodo`）和 payload body（永遠 `true` vs `toggle`）。

**推翻 B**：綠色沒有獨立的 notion 寫入點。
**推翻 C**：綠色 `handleToggleTodo` 明確呼叫 `fetch(..., PATCH)`，不是純 local state。

### 分歧可能在以下更早的層（Step 2 instrumentation 要釐清的）

1. **綠色 fetch 有沒有真的發出 / 收到 2xx**？若 PATCH 500 或 throw，line 604 `throw` 會進 catch，`setClientTodos` 不跑，UI 不會變綠——但使用者說「前端變綠」，暗示 PATCH 回 2xx。那 Notion 就應該寫入了才對……矛盾。
2. **payload body 差異**：綠色送 `{ todoFlag: !currentFlag }`。若 `currentFlag` 因前端 state 不同步永遠是 `true`（已完成），會送 `{ todoFlag: false }`——Notion 設「待辦=false」（按 G17 語意 = pending），視覺上會被使用者誤以為「沒更新」，但其實是**「從打勾翻回空白」**。這個值得 instrument 驗證。
3. **`todoId` 值**：理論上是同一批 Notion page ID，但若綠色 `clientTodos` 拿到的 page.id 帶了前綴 / 被 trim / 編碼差異，API 端 `notion.pages.update({ page_id: params.todoId })` 可能拿錯 page 或失敗。值得打 log 比對。
4. **`body.todoFlag !== undefined`**（route.ts:14）：JSON 解析正常情況下 `true`/`false` 都會通過這個 guard，無 bug 空間。
5. **Notion integration permissions**：若 integration 對「待辦事項」DB 只讀不寫，`notion.pages.update` 會 throw → catch → 500。那 UI 會收到錯誤，line 612 會印 `Toggle todo error:`。使用者可以在 Console 檢查。

## 第五輪前置（使用者回報後要做的判斷）

根因判斷流程：
- Console 有 `[green-todo] PATCH response` `status=200, ok=true` **且** Runtime Logs 有 `[green-todo-api] notion.pages.update returned` **且** Notion 實際狀態對了 → 不是 bug，是使用者誤判
- Console 有 `[green-todo] PATCH response` `status=200` **但** Runtime Logs 沒有對應 `[green-todo-api]` → 打到錯 endpoint / route 沒部署
- Runtime Logs 有 `[green-todo-api] notion.pages.update threw` → Notion API 錯（權限 / payload shape）
- Runtime Logs 有 `[green-todo-api] PATCH received` + `calling notion.pages.update` **但沒有 returned** → Notion SDK hang / timeout
- Console 沒有 `[green-todo] onChange fired` → 前端事件沒觸發（按鈕 binding 出錯）
- `[green-todo] PATCH request` body `{todoFlag: false}` 但使用者以為在打勾 → `currentFlag` 前端 state 不同步
